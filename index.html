Role Studio — configure roles with flexibility, kept in check

Product modules set the ceiling per plant. Roles set the defaults per user. Overrides handle the exceptions — every deviation needs a reason, respects dependencies, and is counted as drift.

The hierarchy — modules cap, roles default, overrides flex

We sell product modules to companies. A module is a sellable slice of the product that unlocks a set of permissions — and each permission still lives in its role type (work execution, approvals & closure, plant visibility…). Modules are switched on per plant under the company's contract.

A user's role + overrides say what they may do; the plant's modules say what exists there. Effective access at a plant = user permission ∧ plant module. Toggle the matrix below — the configurator and the UI previews respect it live.
